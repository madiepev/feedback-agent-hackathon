This is the core challenge. Participants will design AI feedback agents for science writing, test each other's agents, and discover what makes AI feedback effective.

Share Reactions

Phase 2: Agent Design

Build your AI feedback agent using GitHub Copilot instructions

Transition to Phase 2. Teams will discuss strategy, write instruction files, test with Copilot Chat, and commit. 45 minutes total.

Key Design Decisions

Feedback Type

  • Corrections
  • Questions
  • Guidance
  • Mix of all three?

Rubric Usage

  • Score each dimension?
  • Focus on weakest?
  • Evaluate all five?

Tone & Style

  • Coach / Peer / Editor
  • Quote specific text?
  • All at once or step-by-step?
Teams should discuss these decisions before writing code. Encourage diversity of approaches across teams for richer cross-testing later.

Phase 3: Cross-Testing

Evaluate another team’s agent

Announce team assignments: Team 1 tests Team 2, Team 2 tests Team 3, etc. Teams navigate to the assigned repo and test with their own summary.

5 Evaluation Criteria

Understandability

Could you comprehend the feedback?

Specificity

Concrete examples vs. vague statements?

Validity

Accurate and aligned with the rubric?

Actionability

Could you use it to actually improve?

Affective Quality

Did it motivate you to improve?

Walk through the 5 criteria. These are research-based dimensions for evaluating feedback quality. Teams will rate each on a 5-point scale with evidence.

Submit Feedback via GitHub Issue

1

Go to the Main Repository

Not the team’s fork — use the main hackathon repo

2

Create a New Issue

Select the “Agent Feedback (Phase 3)” template

3

Rate & Provide Evidence

Rate all 5 criteria (1–5) with specific examples from the feedback

Submit Feedback Issue
Direct teams to the Issues tab. Emphasize using the template and providing specific evidence for each rating, not just numbers.
20:15 – 20:40

Phase 4: Iteration

Improve based on feedback received

Teams now review the feedback they received and iterate on their agent. Check the Issues tab for feedback about your group.

Common Issues & Fixes

Feedback Received Suggested Fix
“Too vague” Add instructions to quote specific phrases
“Didn’t use the rubric” Require evaluation of each dimension explicitly
“Too harsh” Start with positive observations first
“Not actionable” Require concrete revision suggestions with examples
“Too much at once” Prioritize the 1–2 most important issues first
Find Your Feedback
Show common patterns. Teams should pick 1-3 changes that will have the biggest impact. Test again, iterate until satisfied, then commit v2.
20:40 – 21:00

Phase 5: Community Testing & Reflection

Test multiple agents and share insights

Final phase! Open testing for 14 minutes, then vote and debrief. Encourage teams to test as many agents as they can.

Open Testing & Vote

Explore Freely (14 min)

  • Navigate between teams’ repositories
  • Test agents with YOUR summary
  • Note what works well and why

Vote for the Best (6 min)

  • Go to Discussions: “Which agent was most effective?”
  • Share which team’s agent you liked best
  • Explain why it was effective
Vote in Discussions
Open testing: participants explore freely. At 20:54, direct everyone to the Discussion to vote. Read 2-3 responses aloud for the debrief.
